# プロジェクト概要: MinimalQDA 開発要件定義書

## 1. アプリケーションの目的
「MinimalQDA」は、MaxQDAなどの高機能質的分析ソフトウェアのコア機能（コーディング、メモ、カテゴリー化、可視化）を抽出し、ブラウザ上で10〜30分で素早く分析を完了できる軽量アプリケーションです。学生や初学者をターゲットとします。

## 2. アーキテクチャと制約事項
- **完全フロントエンド完結**: サーバーサイド（バックエンド）やデータベースは一切使用しません。
- **データプライバシー**: 読み込んだファイルやデータは外部に送信せず、すべてブラウザ内（File API等）で処理します。
- **多言語対応**: i18nを用いて、英語(en)と日本語(ja)のUI切り替えを実装します。

## 3. 推奨技術スタック
- **フレームワーク**: React (Vite) + TypeScript
- **スタイリング**: Tailwind CSS + shadcn/ui (またはそれに準ずる軽量なUIライブラリ)
- **状態管理**: Zustand (シリアライズしてファイル保存しやすいため推奨)
- **多言語化**: react-i18next
- **マインドマップ**: React Flow
- **PDF/画像処理**: react-pdf または pdf.js / 画像はCanvas API等による矩形選択

## 4. 画面構成（3ペイン・レイアウト）
- **左ペイン**: (3) 抽出された「コード」のリストと、(4) 「カテゴリー（7色）」の管理エリア。
- **中央ペイン**: (1) 読み込んだデータ（テキストや画像等）の表示と、10色のマーカー（コーディング）を行うメイン作業エリア。
- **右ペイン**: (2) テキストやコードに対して「メモ」を記述・表示するエリア。
- **ヘッダー**: 言語切り替え、ファイルの読み込み、保存、エクスポート、マインドマップ起動ボタンなどを配置。

## 5. コア機能要件
1. **ファイル読み込み**: `.txt`, `.md`, `.xml`, `.pdf`, `.png`, `.jpg` を読み込み表示する。
2. **マーカー機能**: 中央ペインでテキスト（または画像領域）を選択し、10色からマーカーを引く。
3. **メモ機能**: 右ペインで、マーカーを引いた箇所や全体に対してメモを記述する。
4. **コード整理機能**: マーカーを引いた部分のテキストを「コード」として抽出し、左ペインにリスト表示する。
5. **カテゴリー機能**: 左ペインで7色の「カテゴリー」を作成し、抽出したコードをカテゴリーに分類する（ドラッグ＆ドロップ等）。
6. **マインドマップ機能**: 整理したカテゴリーとコードの階層構造を、新しい画面（モーダル等）でマインドマップとして可視化・操作する。
7. **エクスポート機能**: データ・コード・カテゴリーの対応関係が色付きで分かるリスト（HTML出力等）と、マインドマップの画像を出力する。
8. **プロジェクト保存機能**: 現在の全作業状態（ZustandのState等）をJSON形式でシリアライズし、独自拡張子 `.mqda` のファイルとしてローカルにダウンロード保存する。また、そのファイルを読み込んで復元できる。

## 6. 開発フェーズ（Claudeへの指示）
**重要: 一度にすべてを実装しないでください。以下のフェーズに従って、1ステップずつ実装・動作確認を行ってください。各フェーズが完了したら私に報告し、次の指示を待ってください。**

### Phase 1: プロジェクトの初期化とベースレイアウト
- Vite + React + TS環境の構築。Tailwind、i18next、Zustandの導入。
- 日英切り替え機能の実装。
- 左・中央・右の3ペインレイアウトの作成（ダミーデータでUIのみ構築）。

### Phase 2: テキスト対応と状態管理（Zustand）設計
- `.txt`, `.md`, `.xml` の読み込みと中央ペインへの表示。
- テキスト選択による10色マーカー機能の実装。
- マーカー部分を「コード」として左ペインのリストに同期するZustandストアの実装。

### Phase 3: メモ機能とカテゴリー機能
- 右ペインでのメモ機能（コードとの紐付け）の実装。
- 左ペインでの7色カテゴリー作成と、コードの分類機能の実装。

### Phase 4: 保存・復元とエクスポート
- アプリのStateを `.mqda` (実態はJSON) として保存・読み込みする機能。
- 色付きリストのHTMLまたはPDF出力機能の実装。

### Phase 5: マインドマップ機能
- React Flowを用いたマインドマップ画面の実装と、画像エクスポート機能。

### Phase 6: PDF・画像対応（難易度高）
- `.pdf`, `.png`, `.jpg` の読み込み対応。
- テキストハイライトとは異なる、画像に対する矩形選択（バウンディングボックス）によるマーカー機能や、PDFテキストレイヤーのハイライト実装。
